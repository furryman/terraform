# k8s 1.36 Upgrade with Declarative Bootstrap — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the single-node k3s cluster from v1.30.4+k3s1 to v1.36.1+k3s1, bump all in-cluster Helm chart dependencies to current latest stable, and eliminate the runtime `user_data.sh` shell script in favor of fully declarative bootstrap baked into the Packer AMI.

**Architecture:** Two repos change in sequence. `eks-helm-charts` gets chart-dep bumps first (PR A) so we smoke-test against the current 1.30 cluster. Then `terraform` gets the k3s version bump, new Packer-baked declarative bootstrap files (`/etc/rancher/k3s/config.yaml`, `/var/lib/rancher/k3s/server/manifests/*.yaml`), Packer HCL updates, removal of `user_data.sh`, and removal of now-unused Terraform variables. A new AMI is built via GitHub Actions, then `terraform apply` replaces the EC2 instance with one running the new AMI. k3s starts, helm-controller auto-applies the baked manifests (argocd namespace + argo-cd HelmChart CR + App-of-Apps Application), ArgoCD reconciles all workloads from git.

**Tech Stack:** Terraform >= 1.15, AWS provider ~> 6.31, Packer (amazon-ebs plugin ~> 1.3), HashiCorp Configuration Language, k3s (Kubernetes 1.36), helm-controller (k3s built-in, `helm.cattle.io/v1` API), Helm 3.15.4, ArgoCD chart 9.5.15, Gateway API v1.

**Design reference:** `docs/plans/2026-05-24-k8s-1.36-upgrade.md` (approved 2026-05-24).
